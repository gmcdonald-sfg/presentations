Improving your UI

  • shinydashboard Snazzy dashboards
  • shinythemes modify color of default shiny objects
  • shinyBS adds little things shiny doesnt come prepackaged with, i.e., tooltips
  • shinyjs - adds javascript features
  • shinytoastr popup notification messages
  • miniUI build UIs specifically for mobile devices
  • wellPanel - Fixed, hovering panels in UI
  • htmltools - use R programming commands to generate HTML

Improving your UI (continued)

  • req - require certain reactive sources, or stop current chunk - minimize errors
  • debounce and throttle - Great for improving user experience
  • Update family of functions - Dynamically update any element of the UI from server (e.g., updateNumericinput)
  • Drill down reporting

Performance improvements

  • Pre-process as much as possible. If you must, process in global instead of server.
  • Profiling and performance - Nice overview
  • Profvis - Interactive visualizations for profiling code
  • shinytest - Test shiny app functionality
  • async programming using promises - Asynchronous programming - start a long operation, but don't need to wait around for results

Other nifty new things

  • modules - Reuse chunks of commonly used UI and server code
